Multi-Source Data Integration

The hardest numerical reasoning skill — and the one that separates top performers from everyone else. One question, multiple data sources, one answer.

1

What Multi-Source Questions Look Like

In harder SHL and Aon tests, a single question may require you to: (a) read a value from Table 1, (b) read a percentage from a pie chart, (c) apply a growth rate from a text note, and (d) combine them all to calculate the answer. The maths is rarely hard — the challenge is organising the data extraction.

These questions are worth exactly the same marks as simple single-source questions, but they take longer and have more places to go wrong. That makes them the key time-pressure battleground. A systematic method is the only way to handle them consistently.

2

The 5-Step Method

1
Read the Question
What is it actually asking? Underline the key output: "total cost", "percentage growth", "difference between". Know your destination before you start navigating.
2
Identify Your Sources
Which tables, charts, or text notes contain the data you need? Tag each one mentally: "I need headcount from Source A and salary from Source B."
3
Extract from Source 1
Find the specific value. Write it down or type it into your calculator. Check units. Don't move to Source 2 until you've confirmed this number.
4
Extract from Source 2 (and 3...)
Repeat for each additional source. Write each value down separately. Double-check you're reading the right row, column, and time period for each source.
5
Combine and Calculate
Only now do you combine the extracted values. Apply the formula. Check: does the answer make sense? Is it in the right units?
Why Write Values Down?

Under time pressure, your working memory is your weakest link. If you try to hold three extracted values in your head while switching between sources, you will misremember one. Write each value down the moment you extract it — even "obvious" ones. The 5 seconds of writing saves 30 seconds of re-extraction.

Worked Example 1 "What is the total annual salary cost for the Marketing department?"

Step 1: What's being asked?

Total salary cost = headcount × average salary. I need both pieces of data.

Step 2: Which sources?

Source A has headcount. Source B has average salary.

Step 3: Extract from Source A

1Marketing headcount = 45 ← write it down

Step 4: Extract from Source B

2Marketing avg salary = £42k ← write it down

Step 5: Combine and calculate

3Total cost: 45 × £42,000 = £1,890,000 = £1,890k

Trap: Using the wrong department's headcount or salary. Marketing headcount (45) × Technology salary (£62k) = £2,790k — which would likely appear as a wrong-answer option.

Cross-Table Alignment

When two tables share a common variable (like "Department"), always verify you're reading the same row label in both tables. Under pressure, it's easy to read row 3 in Source A but row 4 in Source B if the tables are ordered differently.

Worked Example 2 "What was Europe's Q2 revenue?"

Step 1: What's being asked?

Europe's revenue specifically in Q2. No source gives this directly — it must be calculated.

Step 2: Which sources? Plan the route.

I can get Europe Q3 revenue from Source C (Q3 total) + Source D (Europe %). Then I can reverse the 12% growth from the Note to find Q2.

Step 3: Extract from Source C

1Q3 total revenue (from bar chart): £8,500k

Step 4: Extract from Source D + Note

2Europe's share of Q3 (from pie chart): 40%
3Growth rate Q2→Q3 (from text note): 12%

Step 5: Combine and calculate

4Europe Q3 revenue: 40% × £8,500k = £3,400k
5Europe Q3 = Europe Q2 × 1.12 (12% growth)
6Therefore: Europe Q2 = £3,400k ÷ 1.12 = £3,035.7k ≈ £3,036k

Traps: (1) Using Q2 total revenue (£7,200k) instead of Q3 when calculating Europe's share. (2) Subtracting 12% (£3,400 × 0.88 = £2,992k) instead of dividing by 1.12. Remember from Module 2: reverse percentage = divide by (1 + rate), not multiply by (1 − rate).

The Most Common Multi-Source Mistake

Using the wrong time period from the wrong source. The pie chart shows Q3 breakdown. The bar chart shows all quarters. The text note references Q2→Q3 growth. It's easy to accidentally apply the pie chart percentages to Q2's total instead of Q3's. Always tag each extracted value with its time period: "£8,500k (Q3)" not just "£8,500k".

Practice Question 1 — Two Sources
What was Corporate's annual operating profit? (Use Sources A + B.)

(A) £1,890k

(B) £2,268k

(C) £3,132k

(D) £5,400k

Show Answer ▾

A Corporate revenue: £5,400k. B Corporate cost ratio: 58%.

1Profit = Revenue × (1 − cost ratio) = £5,400k × (1 − 0.58)
2= £5,400k × 0.42 = £2,268k → Answer: B

Trap: Option C (£3,132k) = £5,400 × 0.58 — that's the cost, not the profit. Option D is revenue without subtracting costs at all.

Practice Question 2 — Two Sources
What was Wealth Management's revenue in Q3? (Use Sources C + D.)

(A) £504k

(B) £612k

(C) £648k

(D) £720k

Show Answer ▾

C Q3 total: £3,600k. D Wealth's Q3 share: 18%.

1Wealth Q3: 18% × £3,600k = £648k → Answer: C

Trap: Option A (£504k) = 18% × £2,800k — using the annual revenue from Source A instead of Q3 from Source C. The pie chart (Source D) applies to Q3, not the full year.

Practice Question 3 — Three Sources
What percentage of Corporate's annual revenue was earned in Q3? (Use Sources A + C + D.)

(A) 24.5%

(B) 28.0%

(C) 32.4%

(D) 42.0%

Show Answer ▾

Three sources needed: C for Q3 total, D for Corporate's Q3 share, A for Corporate's annual total.

1Corporate Q3 revenue: 42% × £3,600k = £1,512k (Sources C + D)
2Corporate annual revenue: £5,400k (Source A)
3Q3 as % of annual: £1,512k ÷ £5,400k × 100 = 28.0% → Answer: B

Trap: Option D (42%) is Corporate's share of Q3 total revenue — not Corporate's Q3 as a percentage of its own annual revenue. These are very different questions.

Practice Question 4 — Three Sources (Hard)
Which division generated the highest operating profit in Q3? (Use Sources B + C + D.)

(A) Retail

(B) Corporate

(C) Wealth

(D) Insurance

Show Answer ▾

First get each division's Q3 revenue (C + D), then apply profit margins (B).

Q3 revenues (from C + D): Q3 total = £3,600k

1Retail Q3: 28% × 3,600 = £1,008k
2Corporate Q3: 42% × 3,600 = £1,512k
3Wealth Q3: 18% × 3,600 = £648k
4Insurance Q3: 12% × 3,600 = £432k

Q3 profits (from B): Profit = Revenue × (1 − cost ratio)

5Retail: £1,008 × 0.28 = £282.2k
6Corporate: £1,512 × 0.42 = £635.0k
7Wealth: £648 × 0.35 = £226.8k
8Insurance: £432 × 0.19 = £82.1k

Answer: B — Corporate at £635.0k

Trap: Corporate also has the highest Q3 revenue, so the answer seems obvious. But this isn't always the case — a smaller division with a higher profit margin could win. You must calculate all four to be certain. In a real test, estimate first: Corporate has both the highest revenue share (42%) and the highest profit margin among the big divisions (42%), so it's the clear winner — you could skip the other calculations to save time.

Key Takeaways

  • Use the 5-step method for every multi-source question: read the question, identify sources, extract from each, then combine. Don't skip steps under pressure.
  • Write down each extracted value the moment you find it — including its time period and units. Working memory fails under exam conditions.
  • The most common error is using data from the wrong time period — applying a pie chart's Q3 percentages to Q2's total, or reading the wrong quarter from a bar chart.
  • Tag your extracted values: "£8,500k (Q3, total)" not just "£8,500k". This prevents cross-contamination between sources.
  • For reverse percentages in multi-source questions (like finding Q2 from Q3 + growth rate), divide by (1 + rate). Never subtract the percentage.
  • On the hardest questions requiring 3+ sources, plan your route before extracting any data. Know which calculation chain leads to the answer.
